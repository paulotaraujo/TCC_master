\chapter{Trechos de Código e Referência Operacional}

Este apêndice reúne trechos representativos dos algoritmos centrais e os comandos utilizados para transmitir os registros COMTRADE, controlar a reprodução, configurar a unidade receptora e habilitar as funções de proteção. Nos fragmentos a seguir, comentários com reticências indicam partes omitidas somente para compactação. Os nomes das estruturas e das variáveis foram preservados para permitir sua localização direta nos arquivos \path{embedded_generator.ino} e \path{embedded_receiver.ino}.

\section{Trechos representativos da implementação}

\subsection{Buffer circular da unidade geradora}

O fragmento seguinte mostra a estrutura de 4096 quadros que desacopla a produção dos dados da atualização temporizada do DAC:

\begin{verbatim}
struct FrameRing {
  Frame items[SAMPLE_BUFFER_CAPACITY];
  uint16_t head = 0, tail = 0, count = 0;

  bool push(const Frame &frame) {
    if (count >= SAMPLE_BUFFER_CAPACITY) return false;
    items[head] = frame;
    head = (head + 1) % SAMPLE_BUFFER_CAPACITY;
    count++;
    return true;
  }

  bool pop(Frame &out) {
    if (count == 0) return false;
    out = items[tail];
    tail = (tail + 1) % SAMPLE_BUFFER_CAPACITY;
    count--;
    return true;
  }
};
\end{verbatim}

\subsection{Temporização e detecção de underflow}

A tarefa do DAC acumula os intervalos relativos em uma referência absoluta. Se a fila estiver vazia, registra underflow e suspende a reprodução até que existam novos quadros:

\begin{verbatim}
if (!g_pendingValid) {
  Frame frame;
  bool got = false;
  if (xSemaphoreTake(g_ringMutex, pdMS_TO_TICKS(2))) {
    got = g_frameRing.pop(frame);
    xSemaphoreGive(g_ringMutex);
  }

  if (got) {
    g_pendingFrame = frame;
    g_pendingValid = true;
    g_applyAtUs += g_pendingFrame.dt_us;
  } else {
    if (!g_underflowActive) g_underflows++;
    g_underflowActive = true;
    g_started = false;
  }
}

if ((int32_t)(micros() - g_applyAtUs) >= 0) {
  outputFrame(g_pendingFrame);
  g_pendingValid = false;
}
\end{verbatim}

\subsection{Aquisição e estimação fasorial}

Na receptora, cada canal é obtido pela média de oito conversões. Ao fechamento de um ciclo válido, a DFT fornece os fasores fundamentais e o período entre cruzamentos determina a frequência:

\begin{verbatim}
uint16_t adcV = readAdcAvg(ADC_PIN_V, N_AVG);
uint16_t adcI = readAdcAvg(ADC_PIN_I, N_AVG);

// ... conversao, remocao do offset e aplicacao das escalas

if (g_cycleN >= 8) {
  Phasor v1 = fundamentalPhasorRms(g_cycleV, g_cycleN);
  Phasor i1 = fundamentalPhasorRms(g_cycleI, g_cycleN);
  g_v1Mag = v1.mag;
  g_i1Mag = i1.mag;
  g_v1Angle = v1.angleDeg;
  g_i1Angle = i1.angleDeg;
  g_fEst = 1000000.0f / max(1.0f, (float)periodUs);
}
\end{verbatim}

\subsection{Supervisão direcional das proteções}

A permissão da função 67 é calculada uma vez por ciclo e compartilhada pelas funções habilitadas. O trecho evidencia a formação da condição comum de permissão ou bloqueio:

\begin{verbatim}
bool supervisedPickup = anyMho || oc50PickupNow ||
                        oc51PickupNow || uvPickupNow ||
                        ovPickupNow;

g_dir67Power = g_v1Mag * g_i1Mag *
  cosf((g_v1Angle - g_i1Angle) * PI / 180.0f);

bool forward = g_dir67PrefaultPower > DIR67_POWER_MIN_W;
bool reverse = g_dir67PrefaultPower < -DIR67_POWER_MIN_W;
bool dirOk = DIR67_FORWARD ? forward : reverse;

g_dir67Permit =
  (DIR67_ENABLED && supervisedPickup && dirOk) ? 1 : 0;
g_dir67BlockActive =
  (DIR67_ENABLED && supervisedPickup && !dirOk) ? 1 : 0;
\end{verbatim}

Esses fragmentos documentam a arquitetura algorítmica sem duplicar toda a base de código. As rotinas completas incluem validações, mensagens de diagnóstico, tratamento dos comandos, funções 21, 27, 50, 51 e 59 e atualização das saídas digitais.

\section{Transmissão e reprodução das amostras}

O comando básico da unidade geradora é:

\begin{verbatim}
cd generator/python/embedded_generator
python3 run_embedded_generator.py \
  --cfg samples/sample_3/export.cfg \
  --bdat samples/sample_3/export.bdat \
  --port /dev/ttyUSB0 \
  --baud 921600 \
  --receiver-config config.json
\end{verbatim}

\section{Argumentos das funções de proteção}

\begin{table}[H]
  \centering
  \small
  \caption{Argumentos para habilitação e ajuste das proteções.}
  \begin{tabular}{p{5.5cm}p{7.7cm}}
    \toprule
    Argumento & Ordem e significado dos valores \\
    \midrule
    \texttt{--over-current A B C} & $A$: acréscimo sobre $I_N$ para a função 50 instantânea; $B$: acréscimo para a função 51 temporizada; $C$: atraso da função 51. Por exemplo, \texttt{50 10 0.05} define 50 em $1{,}50I_N$ e 51 em $1{,}10I_N$ por \SI{0,05}{\second}. \\
    \texttt{--distance A B C D} & $A$: módulo da impedância total da linha em ohms; $B$: alcance da zona 1 em porcentagem; $C$: alcance da zona 2 em porcentagem; $D$: atraso da zona 2 em segundos. \\
    \texttt{--distance-line-angle A} & Ângulo da impedância da linha em graus, empregado na característica de distância. \\
    \texttt{--directional-67 A} & $A$: sentido \texttt{forward} ou \texttt{reverse}, definido pelo sinal permitido da potência ativa de pré-falta. \\
    \texttt{--directional-67-power-min A} & Módulo mínimo da potência ativa de pré-falta necessário para validar a decisão direcional; padrão de 10000 nas unidades de engenharia do ensaio. \\
    \texttt{--under-voltage A B C} & $A$: tensão remanescente crítica, em porcentagem de $V_N$, para trip instantâneo; $B$: queda percentual para o estágio temporizado; $C$: atraso. Exemplo: \texttt{50 10 0.20} corresponde a $0{,}50V_N$ instantâneo e $0{,}90V_N$ por \SI{0,20}{\second}. \\
    \texttt{--over-voltage A B C} & $A$: elevação percentual para trip instantâneo; $B$: elevação para o estágio temporizado; $C$: atraso. Exemplo: \texttt{50 10 0.20} corresponde a $1{,}50V_N$ instantâneo e $1{,}10V_N$ por \SI{0,20}{\second}. \\
    \bottomrule
  \end{tabular}
  \legend{Fonte: Autoria própria.}
\end{table}

As opções de proteção são independentes e podem ser combinadas. Quando \texttt{--directional-67} é incluída, a permissão direcional supervisiona as demais funções habilitadas no mesmo ensaio. Quando uma opção é omitida, o grupo correspondente é explicitamente desabilitado durante a configuração, evitando a permanência de ajustes de uma execução anterior.

\section{Sequência operacional recomendada}

\begin{enumerate}
  \item conectar e identificar separadamente as portas seriais das unidades geradora e receptora;
  \item executar \path{run_embedded_generator.py}, aguardar o processamento e confirmar a criação do \texttt{config.json};
  \item no terminal da geradora, enviar \texttt{s} e manter a pré-falta contínua;
  \item executar \path{run_embedded_receiver.py} com o mesmo JSON e com os ajustes desejados;
  \item verificar, pelo comando \texttt{status}, se $V_1$ e $I_1$ em pré-falta correspondem aos nominais do JSON e, se necessário, repetir o comando da receptora com \texttt{--v-scale-correction} e \texttt{--i-scale-correction};
  \item enviar \texttt{f} para aplicar o evento completo e acompanhar as mensagens da receptora;
  \item registrar eventos, trips e mensagens de diagnóstico; e
  \item usar \texttt{q} para retornar ao repouso ou \texttt{x} para encerrar o terminal da geradora.
\end{enumerate}
